Firearms

Firearms

Currently the only—and therefore most developped—system in the game, the firearms are where the fun is for most people; indeed, most concepts introduce at least one new toy to play with, and that's why it's important for the sandbox to do them right.

Prerequisites

  • Everything required in the basic guide
  • Art for your weapon (kinda)

Preparing the images

As of writing this, only the world image and silhouette are really necessary; the loot image isn't used. Navigate to assets/items/firearms and create a new folder. Inside, place all your images. The names don't matter, so long as you remember them for the next step. You'll notice that every other firearm currently in the game has its loot image, silhouette and possibly world image located here. You'll also notice a folder named shared—this is where the stick images are located, used by weapons that don't have custom world images, like the AK-47 or M870.

Now strap in, because we have to look at what each field does. One at a time. Each field is expressed with the following syntax:
(field-name: field-type): description

If a field is optional, there will be a question mark between the field's name and the colon following it.
(optional-field?: field-type): description

Info:
Some basic types:
  • string represents any type of text, always wrapped in double quotes, "like this".
  • number represents any number: whole, decimal, positive or negative. Scientific notation is done by appending an 'e', followed by the desired exponent (2.3 × 104 becomes 2.3e4).
  • boolean represents either true or false, and nothing else.
  • object is a generic catch-all for properties whose values in turn have properties. (ex: if we consider a house to be an object, with multiple properties such as its worth or land area, then its owner property—which represents the person who owns the house—would also be an object with its own properties, like age, name and so on)
  • T | U represents either type T or type U, meaning that the given variable can be of either type (ex: string | number means that something is either a string or a number).
  • T[] is a collection (formally known in Javascript as an array) of values, with all of those values being of type T. (ex: an array of booleans would be notated boolean[])
Literal values can appear too. Consider 0 | "woohoo": this is saying that the attached value is either 0 or the string "woohoo".

  • (name: string): The gun's name
  • (summary: object): A summary of the weapon's main points. Mostly for bots.
    • (class: string): What type of weapon is this? An assault rifle? Shotgun? Look at other existing weapons for an idea of the naming scheme.
    • (engagementDistance: object): Details about the range the bot should use the weapon. (At 1000 units? 2000?)
      • (min: object): The minimum engagement distance. Targets closer than this will make the bot backpedal.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The minimum engagement distance.
      • (max: object): The maximum engagement distance. Targets farther than this will make the bot move towards them.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The maximum engagement distance.
    • (shouldNoslow: boolean): Whether or not the bot should noslow this weapon.
    • (role: "primary" | "secondary"): Whether bots should prioritize using this weapon or save it as a backup.
  • (dual: boolean): Whether or not this is a dual-wielded weapon. Affects rendering.
  • (images: object): Addresses to relevant images; use false for no image. These are the same images referenced in the last step. Note that these addresses are relative to the directory containing the game's main page—they should therefore all start with assets/items/firearms.
    • (loot: string | false): The weapon's loot image. (Currently unused)
    • (held: string | false): The weapon's held image. The sticks are located in the assets/items/firearms/shared directory.
    • (silhouette: string | false): A silhouette of the weapon—whitening the border on the loot image should suffice. (Optional)
  • (tint: string | number[]): A hex color code or an array of three numbers specifying what color the above-specified world image is to be tinted. Custom world images will usually use a value of #FFFFFF (white)
  • (ballistics: object): Information about the projectile the weapon fires
    • (damage: number): The projectile's base damage.
    • (velocity: object): The projectile's velocity, expressed in either units/second or px/second.
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
      • (value: number): The projectile's velocity.
    • (range: object): How far the projectile will travel before despawning.
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
      • (value: number): The projectile's range.
    • (tracer: object): Information about the bullet's tracer.
      • (width: object): The tracer's width, or thickness.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The tracer's width.
      • (height: object): The tracer's height, or length.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The tracer's height (or length).
    • (obstacleMult: number): The number by which this projectile's base damage is multiplied when striking an obstacle.
    • (headshotMult: number): The number by which this projectile's base damage is multiplied when dealing a headshot (15% chance). A value of 1 completely prevents a weapon from headshotting (the 15% chance isn't even rolled and is completely skipped).
    • (fsa: object): Data relating to a weapon's first-shot accuracy (fsa).
      • (enabled: boolean): Whether or not the weapon has the ability for accurate first shots at all.
      • (rechargeTime: object): Data pertaining to the first-shot accuracy's cooldown
        • (givenIn: "s" | "ms" | "RPM"): Whether the provided value is in seconds, milliseconds or in rounds/minute. The latter is quite rare.
        • (value: number): The minimum delay for two consecutive shots to be fully accurate. Guns with FSA disabled typically set this value to 1e10 seconds
    • (falloff: number): The number by which this projectile's base damage is multiplied every 100 units. Typically lower than 1.
    • (projectiles: number): The number of projectiles per shot. Usually 1, except for shotguns.
  • (suppressed: boolean): Whether or not this weapon is suppressed. Affects how tracers are drawn.
  • (caliber: string): Defines the caliber this weapon uses. The game uses this field to fetch the corresponding ammo data from assets/json/ammo.json for properties such as casings and tracer colors.
  • (firingDelay: object): Data pertaining to how fast this weapon can be fired.
    • (givenIn: "s" | "ms" | "RPM"): Whether the provided value is in seconds, milliseconds or in rounds/minute.
    • (value: number): The minimum delay between two consecutive shots, barring quickswitch and other similar techniques.
  • (deployGroup: number): This weapon's deployGroup. Two weapons with identical non-zero deployGroup's cannot be quickswitched. Typically set to 0 for most weapons.
  • (accuracy: object): Data pertaining to the weapon's inaccuracy.
    • (default: object): The weapon's default accuracy when standing still.
      • (givenIn: "degrees" | "radians" | "gradians" | "turns"): Whether the provided value is in degrees, radians, gradians or number of turns.
      • (value: number): The gun's inaccuracy. Draw an isosceles triangle with its "tip" where the projectile spawns—the given value is the angle at that tip.
    • (moving: object): The weapon's accuracy while moving. This is added on to the default value and does not replace it.
      Implemention discrepancy:
      surviv.io and sources that reference it, like the surviv.io wiki, will give "final" accuracy values: the value of moving accuracy is what you get in-game, and is not added onto the gun's default accuracy.
      • (givenIn: "degrees" | "radians" | "gradians" | "turns"): Whether the provided value is in degrees, radians, gradians or number of turns.
      • (value: number): The gun's movement inaccuracy.
  • (moveSpeedPenalties: object): Describes by how much a weapon will slow down players while doing various actions.
    • (active: object): By how much will this weapon slow down its user when active?
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute"). Usually set to "scale" for this field.
      • (value: number): A number that will be added to the player's speed when this weapon is active.
    • (firing: object): By how much will this weapon slow down its user while the user is firing it?
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute"). Usually set to "scale" for this field.
      • (value: number): A number that will be added to the player's speed when this weapon is fired.
  • (imageOffset: object): Where—in relation to the player's center—the gun's world image will be drawn.
    • (perp: object): The offset on the axis perpendicular to the player's aim line.
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute"). Usually set to "scale" for this field.
      • (value: number): By how much the weapon's image is to be displaced along this axis. Negative values are leftwards (from the perspective of the perpendicular axis) and positive values are rightwards.
    • (parr: object): The offset on the axis parallel to the player's aim line.
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute"). Usually set to "scale" for this field.
      • (value: number): By how much the weapon's image is to be displaced along this axis. Positive values are backwards and negative values are forwards, relative to that axis—as of writing, every gun uses a negative value here.
  • (dimensions: object): Info regarding the world image's dimensions.
    • (width: object): The world image's width, or thickness.
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute"). Usually set to "scale" for this field.
      • (value: number): How wide the world image is.
    • (height: object): The world image's height, or length.
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute"). Usually set to "scale" for this field.
      • (value: number): How long the world image is.
    • (layer: 0 | 1 | 2): The layer this weapon should be rendered on. 0 is the default, and means "under both body and hands". 1 is for weapons like the FAMAS, which are drawn over the hands, but under the body. 2 is for weapons like the M79, which are drawn atop both the body and hands.
  • (reload: object): Information about reloading this weapon.
    • (duration: object): Data pertaining to the reload's duration.
      • (givenIn: "s" | "ms" | "RPM"): Whether the provided value is in seconds, milliseconds or in rounds/minute. The latter is quite rare.
      • (value: number): How long this weapon takes to reload.
    • (ammoReloaded: number | "all"): How much ammunition is replenished per reload. "all" fully refills the weapon's magazine, while numeric values replenish that many rounds.
    • (chain: boolean): For reloads that only partially replenish the weapon magazine, this property dictates whether or not these will be chained until the weapon is full. As of writing, every gun this is applicable to has this setting set to true.
  • (altReload?: object): Information about the reload used when this weapon is out of ammo. Currently only used by the Mosin-Nagant.
    • (duration: object): Data pertaining to the reload's duration.
      • (givenIn: "s" | "ms" | "RPM"): Whether the provided value is in seconds, milliseconds or in rounds/minute. The latter is quite rare.
      • (value: number): How long this weapon takes to reload.
    • (ammoReloaded: number | "all"): How much ammunition is replenished per reload. "all" fully refills the weapon's magazine, while numeric values replenish that many rounds.
    • (chain: boolean): For reloads that only partially replenish the weapon magazine, this property dictates whether or not these will be chained until the weapon is full. As of writing, every gun this is applicable to has this setting set to true.
  • (magazineCapacity: object): Info about this weapon's magazine capacity
    • (normal: number): How many rounds this weapon normally holds per magazine.
    • (firepower: number): How many rounds this weapon holds when the user has the firepower perk. (Currently does nothing)
  • (switchDelay: object): Information about the weapon's switch delay. Swtich delay is the amount of time during which a weapon is unable to fire after swapping to that weapon.
    • (givenIn: "s" | "ms" | "RPM"): Whether the provided value is in seconds, milliseconds or in rounds/minute. The latter is quite rare.
    • (value: number): How long this weapon takes to become ready to fire after being switched to. Can be bypassed by quickswitching from another weapon with a different deployGroup or whose deployGroup is 0.
  • (handPositions: object): Where the player's hands are situated when this weapon is active.
    • (leftHand: object): The left hand's position. (0, 0) is the body's center.
      • (perp: object): The left hand's position along the axis perpendicular to the player's aim line.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute"). "scale" is typically used.
        • (value: number): The left hand's position along the perpendicular axis. Negative values are leftwards, positive values are rightwards, both relative to that axis.
      • (parr: object): The left hand's position along the axis parallel to the player's aim line.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute"). "scale" is typically used.
        • (value: number): The left hand's position along the parallel axis. Negative values are forwards, positive values are backwards, both relative to that axis.
    • (rightHand?: object): The right hand's position. (0, 0) is the body's center. If the weapon is held with only one hand, such as the M9, do not include this property; delete it entirely.
      • (perp: object): The right hand's position along the axis perpendicular to the player's aim line.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute"). "scale" is typically used.
        • (value: number): The right hand's position along the perpendicular axis. Negative values are leftwards, positive values are rightwards, both relative to that axis.
      • (parr: object): The right hand's position along the axis parallel to the player's aim line.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute"). "scale" is typically used.
        • (value: number): The right hand's position along the parallel axis. Negative values are forwards, positive values are backwards, both relative to that axis.
  • (projectileSpawnOffset: object): The offset applied to projectiles' spawns, with (0, 0) being the forward-most tip of the player model, relative to the player's aim line's direction.
    • (perp: object): The projectile's spawn offset along the axis perpendicular to the player's aim line.
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
      • (value: number): The projectile's spawn offset along the perpendicular axis. Negative values are leftwards, positive values are rightwards, both relative to that axis.
    • (parr: object): The projectile's spawn offset along the axis parallel to the player's aim line.
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
      • (value: number): The projectile's spawn offset along the parallel axis. Negative values are backwards, positive values are forwards, both relative to that axis.
  • (casings: object):

    As of right now, every gun in the sandbox uses values that only "feel" right—no formal comparisons have been made to surviv.io to verify the exactness of these values.

    • (spawnOffset: object): The offset applied to a casing's spawn location, with (0, 0) being the player's center.
      • (perp: object): The casing's spawn offset along the axis perpendicular to the player's aim line.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The casing's spawn offset along the perpendicular axis. Negative values are leftwards, positive values are rightwards, both relative to that axis.
      • (parr: object): The casing's spawn offset along the axis parallel to the player's aim line.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The casing's spawn offset along the parallel axis. Negative values are backwards, positive values are forwards, both relative to that axis.
    • (velocity: object): The velocity at which a casing is ejected. All values are per second (units/seconds, px/second, etc)
      • (perp: object): The casing's velocity along the axis perpendicular to the player's aim line.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The casing's velocity along the perpendicular axis. Negative values are leftwards, positive values are rightwards, both relative to that axis.
        • (variation: object): Info about variations in velocity.
          • (givenIn: "ratio" | "absolute"): Whether the provided value is relative to the previously provided value for the "value" field ("ratio") or to the world's coordinate system ("absolute").
          • (value: number): The variation in velocity along the perpendicular axis.
          • (plusOrMinus: boolean): Whether or not the variation can apply in both directions; given a value of 3 and a deviation of 2, plusOrMinus: false corresponds to the range [3, 5], while plusOrMinus: true corresponds to the range [1, 5], or 3 ± 2.
      • (parr: object): The casing's velocity along the axis parallel to the player's aim line.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The casing's velocity along the parallel axis. Negative values are backwards, positive values are forwards, both relative to that axis.
        • (variation: object): Info about variations in velocity.
          • (givenIn: "ratio" | "absolute"): Whether the provided value is relative to the previously provided value for the "value" field ("ratio") or to the world's coordinate system ("absolute").
          • (value: number): The variation in velocity along the parallel axis.
          • (plusOrMinus: boolean): Whether or not the variation can apply in both directions; given a value of 3 and a deviation of 2, plusOrMinus: false corresponds to the range [3, 5], while plusOrMinus: true corresponds to the range [1, 5], or 3 ± 2.
      • (angular: object): The casing's angular velocity. Positive values make the casing turn clockwise.
        • (givenIn: "degrees" | "radians" | "gradians" | "turns"): Whether the provided value is in degrees, radians, gradians or number of turns. Turns are typically used.
        • (value: number): The casing's angular velocity.
        • (variation: object): Info about variations in angular velocity.
          • (givenIn: "ratio" | "absolute"): Whether the provided value is relative to the previously provided value for the "value" field ("ratio") or to the world's coordinate system ("absolute").
          • (value: number): The variation in angular velocity.
          • (plusOrMinus: boolean): Whether or not the variation can apply in both directions; given a value of 3 and a deviation of 2, plusOrMinus: false corresponds to the range [3, 5], while plusOrMinus: true corresponds to the range [1, 5], or 3 ± 2.
    • (spawnOn: "fire" | "reload"): When to spawn casings: "fire" spawns a casing when the weapon is fired, "reload" spawns as many casings as the weapon's magazine capacity when a reload is initiated.
    • (spawnDelay: object): Data about when to spawn the casings.
      • (givenIn: "s" | "ms" | "RPM"): Whether the provided value is in seconds, milliseconds or in rounds/minute. The latter is quite rare.
      • (value: number): How long after the action specified in "spawnOn" to spawn the casing(s). Ex: a value of "1s" here and a value of "spawnOn: reload" will make casings eject 1 second after a reload is started.

      • (parr: object): The component of the impulse parallel to the player's aim line.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute"). "scale" is typically used.
        • (value: number): The impulse's parallel component along said axis. Negative values are forwards, positive values are backwards, both relative to that axis.
    • (duration: object): Data pertaining to how long the impulse's animation lasts.
      • (givenIn: "s" | "ms" | "RPM"): Whether the provided value is in seconds, milliseconds or in rounds/minute. The latter is quite rare.
      • (value: number): How long the recoil animation will last.
  • (possibleFireModes: string[]): The modes in which this gun can fire. Currently, all weapons only have one, and if they had more, there'd be no way to cycle through them. Within this array, each element can be any of the following: "automatic" for fully-automatic fire, "semi" for semi-automatic fire, "burst-x"—where x is the number of shots per burst (ex: burst-2)—for burst fire that necessitates releasing the fire button between bursts and "auto-burst-x"—where x is the number of shots per burst (ex: auto-burst-3)—for burst fire where bursts are automatically fired in succession so long as the mouse is held down; burst weapons in surviv.io like the FAMAS and AN-94 use this instead of "burst-x".
    If you decide to include multiple modes, remember to separate them with commas!
  • (burstProps?: object): Information about this weapon's burst mode(s), if applicable.
    • (givenIn: "s" | "ms" | "RPM"): Whether the provided value is in seconds, milliseconds or in rounds/minute. The latter is rare.
    • (value: number): How long the animation will take to complete.
