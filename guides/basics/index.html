The basics

Foreword

Welcome to the guide for modifying surviv.io sandbox! This tool is designed to help conceptors (and anyone with an idea for surviv.io) visualize that idea in a more conrete way in an environment as similar to surviv as possible; while the project is still very much in development, there's still fun to be had. This series of guides will hopefully give you all the tools you need to implement your concepts. Note that some sections may be more technical than others; warnings and other information relevant to proper understanding of these guides will be provided.

Prerequisites

  • A computer (mobile or other such devices will not work)
  • Basic computer literacy, such as knowing how to navigate your hard drive, moving around files, what an address is, etc.
  • (optional) Understanding of—or even fluency in—Javascript and JSON

Getting the code

Currently, the game is hosted on GitHub. However, for us to make lasting changes—or any changes, for that matter—, we must have a local copy of the game on our computer for us to work with. Navigate to the game's repositry on GitHub and clone it. Once cloned, you can move it from your Downloads folder to somewhere more convenient if you wish.

Code editors

While optional, serious modders (or anyone who dislikes headaches) will want to consider installing a dedicated code editor; these software provide syntax highlighting and code completion, two tools to aid in developping mods, even mods as simple as modifying JSON files. Visual Studio Code is a great editor—more or less the gold standard at this point—, and will provide you with everything you need.

// A program for calculating π

function fact(x: number): number {
    if (x < 2) {
        return x || 1;
    }

    return fact(x - 1);
}

const iterationCount: number = 100;

const π = 2 * (new Array<void>(iterationCount))
    .map((v: void, i: number) => fact(i) / fact(fact(2 * i + 1)))
    .reduce((prev: number, curr: number) => prev + curr);

console.log(`π is about ${π}`);

localhost

In order to run the game, your computer needs to host a local server (hence the name). See this MDN article on the topic for the required procedure and justification—MDN is also just an amazing resource for anything related to web development, whether it be Javascript help, CSS stlying syntax, HTML design practices, HTTP guides or whatever else.
