Explosions

Making things explode is fun. In the right hands, explosives can be very powerful, both in real life and in-game. This guide will serve to teach you how to implement your own explosions into the sandbox.

Prerequisites

  • Everything required in the basic guide
  • (optional) If applicable, art for your explosion (custom decal / custom shrapnel)

Preparing the images

If your explosion doesn't use the default decal (assets/decals/map-barrel-res-01.svg) or doesn't use the default shrapnel (assets/items/ammo/tracer.png), then you may import your own. As of writing, no dedicated structure exists to accommodate these assets, so you are free to place them wherever you want, so long as you remember where they are.

Now let's look at each one individually. Each field is expressed with the following syntax:
(field-name: field-type): description

If a field is optional, there will be a question mark between the field's name and the colon following it.
(optional-field?: field-type): description

Info:
Some basic types:
  • string represents any type of text, always wrapped in double quotes, "like this".
  • number represents any number: whole, decimal, positive or negative. Scientific notation is done by appending an 'e', followed by the desired exponent (2.3 × 104 becomes 2.3e4).
  • boolean represents either true or false, and nothing else.
  • object is a generic catch-all for properties whose values in turn have properties. (ex: if we consider a house to be an object, with multiple properties such as its worth or land area, then its owner property—which represents the person who owns the house—would also be an object with its own properties, like age, name and so on)
  • T | U represents either type T or type U, meaning that the given variable can be of either type (ex: string | number means that something is either a string or a number).
  • T[] is a collection (formally known in Javascript as an array) of values, with all of those values being of type T. (ex: an array of booleans would be notated boolean[])
Literal values can appear too. Consider 0 | "woohoo": this is saying that the attached value is either 0 or the string "woohoo".

Note that the first field is explosion's name. See the other explosion in the file for an example.


  • (damage: number): The explosion's damage, not including shrapnel.
  • (obstacleMult: number): A number by which the explosion's and any shrapnel's damage is multiplied when said damage is applied to an obstacle.
  • (radii: object): Data about this explosion's radii. Since the radii of the visual explosion and the damage area don't match, and both have a minimum and maximum, we end up with 4 distinct values.
    • (visual: object): Data about this explosion's "visual" radii: the circle the player sees on-screen.
      • (min: object): The smallest radius the visual component of the explosion; the explosion starts at this radius and grows to its maximum radius.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The minimum visual radius.
      • (max: object): The largest radius the explosion's visual component reaches. Linearly interpolated to from the minimum radius.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The maximum visual radius.
    • (damage: object): Data about this explosion's "damage" radii: the circle within which entities take damage.
      • (min: object): The "max damage" radius, so to speak; entities within the circle delimited by this radius will take full damage from the explosion.
        Possible implementation discrepancy:
        This behavior might differ from surviv.io's.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The "max damage" radius.
      • (max: object): The largest circle within which entities will take damage; entities outside of this circle will suffer no damage from the explosion itself, but may still be hit by shrapnel.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The maximum radius.
  • (lifetime: object): Data pertaining to how long the explosion lasts. Note that this is purely visual: damage is applied when the explosion is spawned, and shrapnel is also spawned as soon as the explosion spawns.
    • (givenIn: "s" | "ms" | "RPM"): Whether the provided value is in seconds, milliseconds or in rounds/minute. The latter is quite rare.
    • (value: number): How long the explosion will last.
  • (color: number[] | string): Purely cosmetic. Defines the "main" color the explosion will fade around. Only three numbers, corresponding to the red, green and blue channels, are needed; extras will do nothing. Alternatively, a hex string is also accepted.
  • (decal: object): Information about the decal left behind by the explosion.
    • (img: string): The path to the decal image. See here.
    • (width: object): The decals's width.
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
      • (value: number): The decal's width.
    • (height: object): The decal's height.
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
      • (value: number): The decal's height. Check the original image to determine its aspect ratio so that you can adjust this value to maintain it.
    • (tint: string | number[]): What color to tint the image pointed to by the address specified above. Either a hex string or an array of three numbers works.
  • (shrapnel: object): Information about the shrapnel created by the explosion.
    • (count: number): The number of shrapnel spawned in when the explosion is created.
    • (damage: number): The damage one piece of shrapnel deals.
    • (color: string | number[]): The color that the shrapnel's image is to be tinted. Either a hex string or three numbers for RGB.
    • (img: string): The path to the shrapnel's image. See here.
    • (velocity: object): The shrapnel's velocity, expressed in either units/second or px/second.
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
      • (value: number): The shrapnel's velocity.
    • (range: object): The shrapnel's range; the distance before which it despawns.
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
      • (value: number): The shrapnel's range.
      • (variation: object): Information about the variation in range.
        • (givenIn: "ratio" | "absolute"): Whether the provided value is relative to the previously provided value for the "value" field ("ratio") or to the world's coordinate system ("absolute").
        • (value: number): The variation in range.
        • (plusOrMinus: boolean): Whether or not the variation can apply in both directions (true) or not (false).
    • (falloff: number): The number by which the shrapnel's damage is multiplied every 100 units. Since shrapnel tends to have ranges way below that, this property is somewhat redundant.
    • (tracer: object): Information about the shrapnel's tracer.
      • (width: object): The tracer's width, or thickness.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The tracer's width.
      • (height: object): The tracer's height, or length.
        • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
        • (value: number): The tracer's height (or length).

And for the Typescript enthusiasts among you, here's the type annotation for that JSON, taken from util.d.ts.

type hexColor = `#${string}`;
type RGBColor = [number, number, number];
type RGBAColor = [number, number, number, number];
type HSLColor = {
    hue: number;
    saturation: number;
    luminosity: number;
};
type HSBColor = {
    hue: number;
    saturation: number;
    brightness: number;
};
type HSLAColor = HSLColor & {
    alpha: number;
};
type HSBAColor = HSBColor & {
    alpha: number;
};
type colorModes = hexColor | RGBColor | RGBAColor | HSLColor | HSLAColor | HSBColor | HSBAColor;
type scaleOrAbsolute = {
    givenIn: "scale" | "absolute";
    value: number;
};
type ratioOrAbsolute = {
    givenIn: "ratio" | "absolute";
    value: number;
};
type angleModes = {
    givenIn: "degrees" | "radians" | "gradians" | "turns";
    value: number;
};
type timeModes = {
    givenIn: "RPM" | "ms" | "s";
    value: number;
};
type variation = {
    variation: ratioOrAbsolute & {
        plusOrMinus: boolean;
    };
};
type explosionData = {
    damage: number;
    obstacleMult: number;
    radii: {
        visual: {
            min: scaleOrAbsolute;
            max: scaleOrAbsolute;
        };
        damage: {
            min: scaleOrAbsolute;
            max: scaleOrAbsolute;
        };
    };
    lifetime: timeModes;
    color: colorModes;
    decal: {
        img: string;
        width: scaleOrAbsolute;
        height: scaleOrAbsolute;
        tint: colorModes;
    };
    shrapnel: {
        count: number;
        damage: number;
        color: colorModes;
        img: string;
        velocity: scaleOrAbsolute;
        range: scaleOrAbsolute & variation;
        falloff: number;
        tracer: {
            width: scaleOrAbsolute;
            height: scaleOrAbsolute;
        };
    };
};

To use your new explosion type, you need to reference its name in an ammo type's explosion_type field. Then, reference that ammo type in a gun's caliber property, and you're done!
