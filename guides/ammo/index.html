Ammunition

Ammunition

Ammunition is another important aspect of the game; without it, guns are useless. While currently somewhat primitive by virtue of being in active development, the sandbox nevertheless lets you control the way ammunition behaves, from its color to casing and type.

Prerequisites

  • Everything required in the basic guide
  • (optional) Art for your ammunition

Preparing the images

As with firearms, it's a good idea to first organize and move our images into an appropriate spot before modifying any JSON files, as the addresses referenced there depend on your placement choice. In the assets/items/ammo directory lives all the assets related to ammo—as of right now, they're mostly casings, except for 40mm, which has a special projectile. You'll also find tracer.png; like the stick images for firearms, this serves as a default, and the vast majority of ammo types use this.

Editing the JSON file

We must now embed the ammo type into the game by embedding it into ammo.json: a file that contains information about every ammo type in the game. Thankfully, the data needed for ammo types is quite minimalistic, unlike the monstrosity that is a firearm's JSON; that being said, we will still go over each field individually to explain what it does. Navigate to assets/json/ammo.json, open it in your text editor of choice, and scroll to the bottom of the file. Find the last leftmost closing curly brace (}) and add a comma after the closing curly brace that precedes it. Then, paste in the template below.

"": {
    "tints": {
        "normal": "",
        "saturated": "",
        "saturated_alt": "",
        "chambered": ""
    },
    "alpha": {
        "rate": 0,
        "min": 0,
        "max": 0
    },
    "spawnVar": {
        "mean": 0,
        "variation": 0,
        "plusOrMinus": false
    },
    "imageOffset": {
        "parr": 0,
        "perp": 0
    },
    "projectileInfo": {
        "type": "bullet",
        "img": ""
    },
    "casing": {
        "img": "",
        "lifetime": {
            "givenIn": "",
            "value": 0,
            "variation": {
                "givenIn": "",
                "value": "",
                "plusOrMinus": false
            }
        },
        "width": 0,
        "height": 0
    }
}

Now let's look at each one individually. Each field is expressed with the following syntax:
(field-name: field-type): description

If a field is optional, there will be a question mark between the field's name and the colon following it.
(optional-field?: field-type): description

Info:
Some basic types:
  • string represents any type of text, always wrapped in double quotes, "like this".
  • number represents any number: whole, decimal, positive or negative. Scientific notation is done by appending an 'e', followed by the desired exponent (2.3 × 104 becomes 2.3e4).
  • boolean represents either true or false, and nothing else.
  • object is a generic catch-all for properties whose values in turn have properties. (ex: if we consider a house to be an object, with multiple properties such as its worth or land area, then its owner property—which represents the person who owns the house—would also be an object with its own properties, like age, name and so on)
  • T | U represents either type T or type U, meaning that the given variable can be of either type (ex: string | number means that something is either a string or a number).
  • T[] is a collection (formally known in Javascript as an array) of values, with all of those values being of type T. (ex: an array of booleans would be notated boolean[])
Literal values can appear too. Consider 0 | "woohoo": this is saying that the attached value is either 0 or the string "woohoo".

Note that the first field is ammo type's name. See the other ammo types in the file for an example.


  • (tints: object): Data about the colors of the tracers guns using this ammo type
    • (normal: string): The regular color of tracers using this ammo type; most shots will be colored this way.
    • (saturated: string): While unused in surviv, in the sandbox, a bonus feature called headshotsUseSaturatedTracers allows tracers belonging to bullets that are headshots to, well, use this special color.
    • (saturated_alt: string): Some ammo types, like 9mm and 12 gauge, don't have saturated tracer colors (well they do, but they're copies of their respective regular colors). In conjunction with another bonus feature, useInterpolatedSaturatedTracers, this field allows headshot tracers to use the color you define here.
    • (chambered: Currently unused by the sandbox, whether or not this color is used is up to whatever perks the player currently has. Usually fully-saturated versions of the normal color.
  • (alpha: object): Data about the tracer's opacity. Only applies to suppressed weapons using this ammo type.
    • (rate: number): A number that the current opacity is multiplied by every game update until it reaches the minimum or maximum opacity. Tracers start at opacity 1. (fully opaque)
    • (min: number): The minimum opacity this tracer can reach.
    • (max: number): The maximum opacity this tracer can reach. Do not set this value lower than 1.
  • (spawnVar: object): Used uniquely by shotguns as of writing, these properties define how much variation can occur in where this ammo type's bullets spawn. All measurements are given in relation to the player size. ("scale" as opposed to "absolute") If one draws a circle with radius mean centered around the bullet's regular spawn position, the circle corresponds to the bullet's possible spawn locations.
    • (mean: number): The average radius of the aforementioned circle.
    • (variation: number): By how much the above value can change. Given relative to player size and not relative to the mean.
    • (plusOrMinus: number): Whether the above variation only applies in one direction (false) or in both (true).
  • (imageOffset: object): Describes the offset a bullet's image will be drawn relative to its hitbox (the hitbox is always located at the bullet's "true" position). All measurements are given in relation to the player size. ("scale" as opposed to "absolute")
    • (parr: number): The offset along the axis parallel to the bullet's trajectory; positive is backwards, negative is forwards.
    • (perp: number): The offset along the axis perpendicular to the bullet's trajectory; positive is right and negative is left.
  • (projectileInfo: object): See below.
  • (casing: object): Describes the casing guns using this ammo type eject; while the firearms controls the spawn offset and velocity, the ammo type controls the image, lifetime and size.
    Implementation discrepancy:
    These values have not been validated through comparisons with surviv.io, and are just what "look right".
    • (img: string): A path to the casing's image. If you put the images in the recommended directory, your path should start with assets/items/ammo.
    • (lifetime: object): Data about how long this casing lives for before despawning.
      • (givenIn: "s" | "ms" | "RPM"): Whether the provided value is in seconds, milliseconds or in rounds/minute. The latter is quite rare.
      • (value: object): The amount of time after spawning that this casing will disappear.
      • (variation: object): Info about variations in despawn time.
        • (givenIn: "ratio" | "absolute"): Whether the provided value is given relative to the previously provided value for the "value" field ("ratio") or given in milliseconds ("absolute").
        • (value: number): The variation in despawn time.
        • (plusOrMinus: boolean): Whether or not the variation can apply in both directions (true) or only one (false).
    • (width: number): The casing's width, given in absolute units.
    • (height: number): The casing's height (or length), given in absolute units. Consult your image to find its aspect ratio; that way, you can tweak these values to maintain it.

The projectileInfo field

Since ammo types (more specifically 40mm) can spawn in explosions and generally do things other than be solid projectiles, this must be accounted for in the JSON data. However, because each role requires different things—a regular bullet doesn't have a blast radius, for example—it'd be unwise to simply have one catch-all template for ammo types. For this reason, it's worth dedicating an entire section to the projectileInfo field, as it will become (though currently isn't really) too complex for a simple one to two line explanation.

That being said, some properties are shared by all of them.

  • (img: string): The address of the projectile's "world" image. Most ammo types use tracers, and therefore have this property set to assets/items/ammo/tracer.png.
  • (spinVel?: string): Information about how this projectile spins. May be omitted if the projectile does not spin, as is the case for the majority of them.
    • (givenIn: "degrees" | "radians" | "gradians" | "turns"): Whether the provided value is in degrees, radians, gradians or number of turns. Turns are typically used.
    • (value: number): The projectile's angular velocity. Positive values make the projectile turn clockwise.

The type property will dictate what other properties appear on this object.

  • "type": "bullet"

    The simplest. No additional properties required, it speaks for itself. A type of "bullet" indicates that this projectile will self-destruct on collision, applying damage to whatever it collided with.

  • "type": "explosive"

    Indicates that, when certain conditions to be explained shortly are met, this projectile will spawn an explosion.

    • (explosionType: string): A string containing the name of an explosion defined in explosions.json.
    • (heightPeak: number): Used to affect the zoom effect imparted on the projectile's image.
    • (explodeOnContact: boolean): Whether this projectile will spawn its explosion when colliding with an object or player.
    • (maxDist: object): The maximum distance this projectile will travel before detonating.
      Warning:
      surviv.io tends to favor the use fuse times as opposed to maximum range; the sandbox will soon adopt this as well.
      • (givenIn: "scale" | "absolute"): Whether the provided value is relative to the player's size ("scale") or to the world's coordinate system ("absolute").
      • (value: number): The maximum distance.

For those familiar with Typescript's syntax, what follows is the complete type annotation for an ammo type, taken from util.d.ts.

type scaleOrAbsolute = {
    givenIn: "scale" | "absolute";
    value: number;
};
type ratioOrAbsolute = {
    givenIn: "ratio" | "absolute";
    value: number;
};
type angleModes = {
    givenIn: "degrees" | "radians" | "gradians" | "turns";
    value: number;
};
type timeModes = {
    givenIn: "RPM" | "ms" | "s";
    value: number;
};
type offsetData = {
    perp: scaleOrAbsolute;
    parr: scaleOrAbsolute;
};
type variation = {
    variation: ratioOrAbsolute & {
        plusOrMinus: boolean;
    };
};
type ammoData = {
    tints: {
        normal: string;
        saturated: string;
        saturated_alt?: string;
        chambered: string;
    };
    alpha: {
        rate: number;
        min: number;
        max: number;
    };
    spawnVar: {
        mean: number;
        variation: number;
        plusOrMinus: boolean;
    };
    imageOffset: {
        parr: number;
        perp: number;
    };
    projectileInfo: ({
        type: "explosive";
        explosionType: string;
        explodeOnContact: boolean;
        maxDist: scaleOrAbsolute;
        heightPeak: number;
    } | {
        type: "bullet";
    }) & {
        img: string;
        spinVel?: angleModes;
    };
    casing: {
        img: string;
        lifetime: timeModes & variataion;
        width: number;
        height: number;
    };
};

And that's just about it for this guide. You've now implemented your own ammo type into the sandbox successfully. Now try making a gun that uses it, or, if you've already done that, go take your new toy for a test drive!
